排序算法总结

总结常用排序算法，针对某些排序算法的特殊设计思想，进行拓展延伸。

1、冒泡排序

从头到尾遍历数组，每次遍历过程中比较相邻元素，若两元素不满足大小顺序则进行交换，继续与后续元素比较。

时间复杂度：$O(N^2)$，空间复杂度：$O(1)$

2、选择排序

从头到尾遍历数组$n$，第一次遍历寻找最小值对应的下标（与第一个元素交换），第二次遍历寻找次小值对应的下标（与第二个元素交换），依此类推，每次寻找对应小值的下标。

时间复杂度：$O(N^2)$，空间复杂度：$O(1)$

3、插入排序

在要排序的一组数中，假定前n-1个数已经排好序，现在将第n个数插到前面的有序数组中，使得前n个数也是排好序的。

时间复杂度：$O(N^2)$

4、快排

核心思想是让数组中的每个元素找到它的位置，使得所有比它小的数都放置在它的左边，所有比它大的数都放置在它的右边。

时间复杂度：$O(N\log N)$

5、堆排序

利用小/大根堆（升序采用大根堆，降序采用小根堆），首先构造小/大根堆，然后将根元素（最小值/最大值）与末尾元素交换，除末尾元素外的之前元素再重新构造小/大根堆（相当于将根元素推出），然后再将根元素与次末尾元素交换，一直到小/大根堆中所有元素都推出。

时间复杂度：$O(N\log N)$

6、希尔排序

插入排序的一种变种，在要排序的一组数中，根据某一增量分为若干子序列，并对子序列分别进行插入排序。然后逐渐将增量减小，并重复上述过程。直至增量为1，此时数据序列基本有序，最后进行插入排序。

7、归并排序

归并排序是采用分治法的典型应用，通过先递归分解数列，再基于各个小数列进行排序，最终进行数列合并。

时间复杂度：$O(N\log N)$, 稳定排序

8、基数排序

属于分配式排序，又称为“桶子法”，主要是通过键值的部分咨询，将要排序的元素分配到“桶”中，每次按照基数重新将现有桶内数重新分配到桶中。

时间复杂度： $O(N\log (r)m)$，其中$r$为采取的基数,$m$为堆数（桶数）

Software Testing

Organize some knowledge of software testing from coursera. (This blog use English writing to improve my English writing ability)
Reference：Introduction to Software Testing Edition 2 (by Paul Ammann and Jeff Offutt)

Definition of Bug

  • Software Fault: A static defect in the software.
  • Software Error: An incorrect internal state that is the manifestation of some faults.
  • Software Failure: External, incorrect behavior with respect to the requirements or other description of the expected behavior.

PIE Model

PIE Model is used to describe three necessary conditions for observing a failure.

  • Execution/Reachability: The location or locations in the program that contain the fault must be reached.
  • Infection: The state of the program must be incorrect.
  • Propagation: The infected state must propagate to cause some output of the program to be incorrect.

Tips:

  1. A test may not execute the location of the fault!
  2. A test executing the fault may not produce an error!
  3. An error may not be propagated to the output!

Software Tesing Terminology

Test Case: including Test Input(Test Data), Test Oracle(Expected Output), Others(Environment)

Testing vs. Debugging

  • Tesing is to reveal a bug by executing test and observing failure.
  • Debugging is to fix a bug by locating, understanding and correcting fault.

Verification vs. Validation

  • Validation: The assurance that a product, service, or system meets the needs of the customer and other identified stakeholders. It often involves acceptance and suitability with external customers.
  • Verification: The evaluation of whether or not a product, service, or system complies with a regulation, requirement, specification, or imposed condition. It is often an internal process.

Black-box Testing vs. White-box Testing

  • Black-box Testing without source code
  • White-box Tesing with source code
  • Gray-box Tesing (with decompile)

Testing Level
System testing -> Integration testing -> Module testing -> Unit testing

• 原生支持文档存储（JSON样式）、XML和键值存储
• 支持同步、异步复制
• 支持Python、PL、Perl等脚本语言操作
• 支持全文搜索		 1、相对其他关系数据库来说，PostgreSQL是更好的选择（比如以位置为基础的应用，需要大量数组操作的应用等）；
2、面向混合数据模型（需要部分NoSQL功能）需求时，PostgreSQL是较好的选择，因为原生支持JSON/XML/Key-Value存储。		 1、当数据模型不是关系型或者有非常特定的体系结构需求时，PostgreSQL不适用；

数学猜想

记录相关数学猜想，不仅仅局限于数论范畴。

哥德巴赫猜想

猜想内容： 任意大于2的偶数都可以表示成两个素数之和，即所谓的“1+1=2”猜想。例如，4 = 2 + 2，10 = 3 + 7，20 = 7 + 13，…

费马猜想

猜想内容： 当整数$n>2$时，关于$x,y,z$的方程$x^n+y^n=z^n$没有正整数解。
费马猜想现在已被证明！

冰雹猜想（角谷猜想）

猜想内容： 对于任意一个非零整数n，如果n为奇数则乘以3倍加1，如果n为偶数则除以2，反复迭代上述运算，那么n最终将落入4->2->1，n总是会变为1。

Bachet猜想（拉格朗日四平方定理）

猜想内容： 对于任意一个正整数n，都可以表示成至多4个整数的平方和。即对于任意正整数$n$，都能找到4个整数$a_1$，$a_2$，$a_3$，$a_4$，使得$n=a_1^2+a_2^2+a_3^2+a_4^2$，例如$192=8^2+8^2+8^2+0^2$，$23=3^2+3^2+2^2+1^2$。
Bachet猜想已被证明！
Tips 正整数可以表示为三个整数平方和的一个特殊条件为：$n \ne 4^k(8m+7)\Longleftrightarrow n=a_0^2+a_1^2+a_2^2$，也就是说如果$n=4^k(8m+7)$，那么$n$只能表示为4个非0整数的平方和

同余式

设$m$是给定的一个正整数，$a$、$b$是整数，若满足$m|(a-b)$，则称$a$与$b$对模$m$同余，记为$a\equiv b(mod m)$，或记为$a\equiv b(m)$。这个式子成为模$m$的同余式。

Tips 程序计算中余数是可以为负数的，如下例子：

  1. -98 / 10 = -9; -98 % 10 = -8
  2. 98 / 10 = 9; 98 % 10 = 8
  3. 98 / -10 = -9; 98 % -10 = 8

那么当快慢指针相遇时，慢指针走过的距离F+a，快指针走过的距离F+a+b+a，则2(F+a) = F+a+b+a，则F=b，所以求取环的开始节点可以如此操作：通过快慢指针找到相遇的点，新设指针指向链表开头，慢指针和新指针迭代指向后继节点，当两个指针再相遇，即使环的开始位置。
